# run.sh

A Makefile replacement written in pure bash. No DSL, no runtime dependencies,
no build step — just scripts.

## Install

```sh
curl -fsSL https://raw.githubusercontent.com/imgproxy/run-sh/main/install.sh | bash
```

This sets up, in the current project:

- `run` — the task dispatcher (regenerated by the installer; do not hand-edit)
- `.runrc` — shared helpers and terminal color helpers, sourced once before any task runs (yours to edit)
- `bin/*.sh` — one file per task

If `./bin` already exists, the installer will prompt you to choose a different
folder for task files.

## Usage

```sh
./run              # list available tasks
./run help <task>  # show a task's usage
./run <task> [args...]
./run new <task>   # scaffold a new task file
```

## Optional: global command

```sh
curl -fsSL https://raw.githubusercontent.com/imgproxy/run-sh/main/install.sh | bash -s -- --global
```

Installs `run` to `~/.local/bin`, which walks up from your current directory
to find the nearest run.sh project (like `git` finds `.git`), so you can run
`run <task>` from any subdirectory instead of `../../run <task>`.

## Writing tasks

A task is a file at `bin/<name>.sh` defining three functions:

```sh
#!/usr/bin/env bash
# Sourced by ./run — do not execute directly.

description() { echo "One-line summary shown in ./run's task list"; }

help() {
  cat <<'EOF'
Usage: ./run <name> [args...]
EOF
}

main() {
  echo "task logic goes here, receives \"\$@\""
}
```

Scaffold one with `./run new <name>`, or see `examples/` for zero-arg,
positional-arg, flag-parsing, and task-composition patterns.

## Philosophy

- No DSL to learn — it's bash, all the way down.
- No runtime dependency, no build step, no framework.
- `run` is generated and regenerable; your logic lives in `bin/` and `.runrc`,
  which re-installing never touches.
- Compose tasks with `depends_on task1 task2` in a task's `main()`.
